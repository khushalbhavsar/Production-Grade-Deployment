## Component Descriptions

### 1. Source Control (GitHub)

- Repository hosting
- Feature branch management
- Pull Request workflow
- Webhook triggers for CI

### 2. CI Server (Jenkins)

- Multibranch Pipeline
- Automatic branch detection
- Docker image building
- Registry push operations

### 3. Container Registry (DockerHub)

- Docker image storage
- Version tagging
- Image distribution

### 4. Container Orchestration (AWS EKS)

- Managed Kubernetes cluster with node groups
- VPC with public/private subnets
- IAM roles, policies, and OIDC provider for IRSA
- Multi-environment support (dev and prod)

### 5. GitOps (Argo CD)

- Git-based deployments
- Automatic synchronization
- Rollback capabilities
- Drift detection

### 6. Monitoring & Observability

- Kubernetes cluster
- Pod management
- Service load balancing
- Auto-scaling

### 7. Monitoring & Observability

- **Prometheus**: Metrics collection, alerting rules, scrape configs
- **Grafana**: Dashboards, visualizations, alert notifications
- **kube-state-metrics**: Kubernetes object metrics
- **node-exporter**: Node-level system metrics

## Data Flow

1. **Development**: Developer creates feature branch and commits code
2. **PR Creation**: Pull Request triggers Jenkins CI pipeline
3. **CI Pipeline**: Tests run, Docker image built and pushed
4. **Manifest Update**: K8s deployment manifest updated with new image tag
5. **GitOps Sync**: Argo CD detects change and syncs to cluster
6. **Deployment**: New pods deployed with rolling update strategy
7. **Monitoring**: Prometheus scrapes metrics, Grafana displays dashboards
8. **Access**: Users access application via LoadBalancer URL

## Security Considerations

- Secrets stored in Kubernetes Secrets
- Docker credentials in Jenkins credentials store
- GitHub tokens for authenticated operations
- Network policies for pod-to-pod communication
- RBAC for Kubernetes access control
- **IRSA (IAM Roles for Service Accounts)** for pod-level AWS access
- **Private subnets** for EKS worker nodes
